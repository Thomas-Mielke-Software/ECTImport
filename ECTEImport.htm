Überblick

Das CSV-Import-Plugin für EasyCash&Tax bietet die Möglichkeit Buchungen zu importieren die aus anderen Anwendungen stammen. Die Daten müssen in Textdateien die dem CSV-Format entsprechen vorliegen.

 

CSV-Dateiformat

Im folgenden wird der Aufbau einer CSV-Datei (Comma Separated Values) beschrieben.

Jeder Datensatz ist durch eine Zeilenschaltung (CR/LF) getrennt sein. Felder sind durch Kommas getrennt und können optional in Anführungszeichen eingeschlossen sein. Anführungszeichen, die Teil des Feldinhaltes sind, stehen zweimal hintereinander (““), andernfalls werden sie als Begrenzungszeichen interpretiert. Kommas, die nicht von Anführungszeichen eingeschlossen sind, werden als Trennzeichen behandelt. Zwei aufeinanderfolgende Kommas gelten als leeres Feld. Leerzeichen, die unmittelbar vor oder nach einem Trennzeichen stehen, werden ignoriert.

Beispiel

Angenommen ein Datensatz (Zeile) sieht folgendermaßen aus

"Feld 1","Feld"" 2","Feld 3",“Feld 4“

So wird er in folgende Felder zerlegt:

Feld 1

Feld 2

Feld 3

Feld 4

 

Anmerkungen zur Konvertierung von Feldwerten

Im folgenden erhalten Sie einige technische Hinweise, wie die Feldwerte in die einzelnen Datenformate überführt werden.

Text (Felder Beschreibung und Konto)

Führende und nachfolgende Leerzeichen werden entfernt.

 

Datum (Feld Datum)

Bei Datei mit zwestelligen Jahresangaben wird das Jahrhundert folgendermaßen ermittelt: Jahreszahlen zwischen 1 und 37 werden dem aktuellen Jahrtausend (also 2000) zugeordnet, andere Jahreszahlen werden den vergangenen Jahrtausend zugeordnet.

Datumsangaben müssen zwischen dem 1. Januar 1970 und dem 18. Januar 2038 liegen.

 

Betrag (Felder Steuersatz und Bruttobetrag)

Führende und nachfolgende Leerzeichen dürfen vorkommen und werden ignoriert.

 

Ganzzahl (Felder AfA akt. Jahr und AfA Dauer)

Führende und nachfolgende Leerzeichen dürfen vorkommen und werden ignoriert. Leere Felder werden nicht berücksichtigt.

 Dialog Import

Im folgenden werden die Bedeutung der einzelnen Steuerelemente beschrieben.

 

- Importbeschreibung

Diese Liste enthält alle vorhandenen Importbeschreibungen. Wählen Sie die aus, die Sie bearbeiten oder ausführen möchten. Alle weiteren Angaben (Dateiname, Trennzeichen, Zuordnung, etc.) beziehen sich auf die ausgewählte Importbeschreibung.

- Neu

Legt eine neue Importbeschreibung an. Der Dialog Neue Importbeschreibung wird geöffnet und Sie können einen Namen angeben.

- Löschen

Löscht – nach Rückfrage ‑ den im Feld Importbeschreibung ausgewählten Eintrag einschließlich aller damit verbundenen Informationen.

- Dateiname, Durchsuchen...

Geben Sie hier den vollständigen Pfadnamen der Datei an, die importiert werden soll. Über den Schalter Durchsuchen... können Sie einen Dateinamen auswählen.

Wenn Sie lieber mit der Maus arbeiten, können Sie aus dem Windows® Explorer die gewünschte Datei in dieses Fenster ziehen und ablegen. Es wird dabei keine neue Importbeschreibung erstellt, sondern nur der Name der Datei in das Feld Dateiname übernommen.

- Trennzeichen

Geben Sie hier an, durch welches Zeichen die einzelnen Felder in der Datei getrennt sind. Sollten die Felder durch Tabulatorzeichen getrennt sein, geben Sie hier <Tab> (einschließlich der spitzen Klammern) ein.

- Anzahl Kopfzeilen

Die hier eingegebene Anzahl Zeilen wird beim Import überlesen und ihr Inhalt wird nicht importiert.

- Anzeigen

Zeigt den Inhalt der Datei gemäß der oben angegebenen Parameter (Dateiname, Trennzeichen und Anzahl Kopfzeilen) in der Liste Dateiinhalt an.

- Dateiinhalt

Hier werden die ersten zehn Zeilen der Datei angezeigt.

Beachten Sie bitte, daß sich die Anzeige nicht automatisch aktualisiert. Sollten Sie z. B. die Anzahl der Kopfzeilen ändern, müssen die erneut die Schaltfläche Anzeigen drücken um das Resultat zu sehen.

- GAWK Script, Durchsuchen....

Der Pfadname einer GAWK-Skriptdatei. Mehr dazu unter Arbeiten mit GAWK-SkriptenIDH_STEPBYSTEP. Wenn Sie lieber mit der Maus arbeiten, können Sie aus dem Windows® Explorer die gewünschte Skriptdatei in dieses Fenster ziehen. Wenn die Endung der Datei ‚.awk’ ist, wird die Skriptdatei auch in das richtige Feld eingetragen, ansonsten in das Feld für die CSV-Datei.

- Bearbeiten… (GAWK Script)

Öffnet den Texteditor zum Erstellen oder Bearbeiten einer GAWK-Skriptdatei. Mehr dazu unter Arbeiten mit GAWK-SkriptenIDH_STEPBYSTEP.

- Ausführen… (GAWK Script)

Führt eine GAWK-Skriptdatei aus. Dabei wird die angegebene CSV-Datei verarbeitet und das Resultat in eine Datei desselben Namens, nur mit der zusätzlichen Endung ’.awked’ gespeichert. Das Feld Dateiname weiter oben im Dialogfenster wird entsprechend geändert. Zuletzt wird der Inhalte dieser Datei automatisch angezeigt. Mehr dazu unter Arbeiten mit GAWK-SkriptenIDH_STEPBYSTEP.

Zuordnung der Buchungsfelder zu Dateifeldern

Legen Sie hier fest, welches Feld aus der zu importierenden Datei in welches Feld des Buchungssatzes übertragen werden soll.

Ordnen Sie Felder zu, indem Sie die Nummer oder den Namen des Feldes aus der Datei in das entsprechende Eingabefeld (Ein/Aus, Datum, Betrag, etc) eintragen.

- Ein/Aus, Einnahme, Ausgabe

Sofern in der zu importierenden Datei ein Feld für die Kennzeichnung der Zeile als Einnahme oder Ausgabe vorhanden ist, tragen Sie dessen Nummer oder Namen im Feld Ein/Aus ein. In diesem Fall müssen Sie auch die Felder Einnahme und Ausgabe füllen. Tragen Sie im Feld Einnahme den Wert ein, der eine Zeile als Einnahme kennzeichnet (z. B. E). Mit dem Feld Ausgabe verfahren Sie ebenso.

Lassen Sie die Felder Ein/Aus, Einnahme und Ausgabe leer, so wird die Zeile automatisch als Einnahme oder Ausgabe erkannt. Hierzu wird das Feld Betrag verwendet: Negative Werte werden als Ausgaben angesehen und positive Werte werden den Einnahmen zugeordnet.

- Datum, Format, Trennzeichen

Geben Sie im Feld Datum an, in welchem Feld das Buchungsdatum steht. Über die Felder Format und Trennzeichen legen Sie fest, wie das Datum in der Datei steht. Das Feld Trennzeichen legt fest durch welches Zeichen die Angaben für Tag, Monat und Jahr voneinander getrennt sind. Es kann leer bleiben wenn kein Trennzeichen verwendet wird; jedoch geht das Programm dann davon aus, daß Tag- und Monatsangabe zweistellig vorliegen.

Lassen Sie das Feld Datum leer, so erhält jede Buchung das heutige Datum.

Folgende Formate sind derzeit verfügbar:

-          Tag, Monat, Jahr zweistellig (z. B. 21.6.83)

-          Monat, Tag, Jahr zweistellig (z. B. 6.21.83)

-          Jahr zweistellig, Monat, Tag (z. B. 83.6.21)

-          Tag, Monat, Jahr vierstellig (z. B. 21.6.1983)

-          Monat, Tag, Jahr vierstellig (z. B. 6.21.1983)

-          Jahr vierstellig, Monat, Tag (z. B. 1983.6.21)

- Betrag, Format

Tragen Sie im Feld Betrag ein, welches Feld in der Importdatei den Betrag enthält. Diese Angabe darf nicht fehlen. Im darauffolgenden Feld Format legen Sie fest, wie der Betrag in der Datei abgelegt ist.

Folgende Formate sind derzeit verfügbar:

-          Dezimalkomma – z. B. 1.234,56

-          Dezimalpunkt – z. B. 1,234.56

-          Währungseinheit / 100  (z.B. Cent) – z. B. 123456 = 123 Euro und 56 Cent

- Brutto/Netto, Format

Legen Sie hier fest, ob es sich bei dem oben genannten Betrag um einen Brutto- oder Nettobetrag handelt. Sofern in der zu importierenden Datei ein Feld für die Kennzeichnung des Betrages vorhanden ist tragen Sie dessen Namen oder Nummer unter Brutto/Netto ein. In diesem Fall müssen Sie auch die Eingabefelder Brutto und Netto füllen. Tragen Sie im Feld Brutto den Wert ein, der eine Zeile als Einnahme kennzeichnet (z. B. B). Mit dem Eingabefeld Netto verfahren Sie ebenso.

Sind in der zu importierenden Datei nur Bruttobeträge oder nur Nettobeträge vorhanden können Sie unter Brutto/Netto anstelle eines Feldes die Optionen (Nur Bruttobeträge) oder (Nur Nettobeträge) auswählen,

- Beschreibung

Geben Sie im Feld Beschreibung an, in welchem Feld die Beschreibung (Buchungstext) steht. Diese Angabe ist optional.

- Belegnummer

Geben Sie im Feld Belegnummer an, in welchem Feld die Belegnummer steht. Diese Angabe ist optional und erst ab Version 1.16 von EasyCash&Tax verfügbar; in älteren Versionen ist dieses Feld inaktiv.

- Konto

Geben Sie im Feld Konto an, in welchem Feld die Kontobezeichnung steht. Diese Angabe ist optional. Buchungen ohne Konto-Angabe können Sie jedoch nur sehen, wenn Sie das Journal nach Konten anzeigen lassen (Menübefehl Ansicht|Journal nach Konten). Sie werden unter der Kontobezeichnung "-- [noch keinem Konto zugewiesene Einnahmen/Ausgaben] –" geführt.

- Steuersatz, Format

Tragen Sie im Feld Steuersatz ein, welches Feld in der Importdatei den Steuersatz enthält. Diese Angabe ist optional. Im darauffolgenden Feld Format legen Sie fest, wie der Betrag in der Datei abgelegt ist (siehe auch Betrag).

- AfA akt. Jahr

Geben Sie hier das Feld an, in dem das aktuelle AfA-Jahr (im wievielten Jahr das WG genutzt wird) enthalten ist. Ist der Inhalt Null oder leer wird er nicht berücksichtigt. Diese Angabe ist optional.

- AfA Dauer

Geben Sie hier das Feld an, in dem die gesamte Abschreibungsdauer (in Jahren) steht. Sofern Sie dieses Feld zuordnen, muß bei Buchungen für die keine AfA möglich/nötig ist der Inhalt Null oder leer sein. Diese Angabe ist optional.

- AfA Restwert, Format

Geben Sie hier an, in welchem Feld der Restwert steht. Diese Angabe ist optional. Wenn Sie kein Feld zuordnen wird der Restwert berechnet. Im darauffolgenden Feld Format legen Sie fest, wie der Betrag in der Datei abgelegt ist (siehe auch Betrag).

- AfA deg. Satz, Format

Geben Sie hier an, in welchem Feld der Prozentsatz für degressive Abschreibung steht. Diese Angabe ist optional.

Sofern ein Wert größer Null importiert wird, wird die Abschreibung auf Degressiv eingestellt. Im darauffolgenden Feld Format legen Sie fest, wie der Betrag in der Datei abgelegt ist (siehe auch Betrag).

Die Berechnung der degressiven AfA wird erst in einer kommenden Version von EasyCash&Tax unterstützt werden.

- AfA Genauigkeit

Geben Sie hier an, in welchem Feld die Art der AfA kodiert ist. Mögliche Werte sind 0 für Ganzjahres-AfA (z.B. bei Sammelposten bzw. GWG-Pool), 1 für halbjährliche Genauigkeit, 2 für monatsgenaue AfA, bei 3 wird die AfA entsprechend der Programmeinstellungen berechnet. Diese Angabe ist optional.

- Betrieb

Geben Sie hier an, in welchem Feld der Betrieb steht. Wenn Sie nicht mit Betrieben arbeiten, lassen Sie das Feld ohne Zuweisung. Betriebe sind nützlich für alle Steuerzahler, die mehrere EÜRs abgeben müssen, nach Betrieben getrennt. Betriebe können im EC&T-Hauptprogramm unter Menp->Ansicht->Journal nur für Betrieb… angelegt werden.

- Bestandskonto

Geben Sie hier an, in welchem Feld das Bestandskonto steht. Wenn Sie nicht mit Bestandskonten (z.B. Bank, Kasse, PayPal) arbeiten, lassen Sie das Feld ohne Zuweisung. Bestandskonten sind nützlich, wenn man seine Konten kontrollieren möchte um keine Ausgabe zu vergessen, sind aber für die vereinfachte Buchführungspflicht nicht erfoderlich. Bestandskonten können im EC&T-Hauptprogramm unter Menp->Ansicht->Journal nur für Bestandskonto… angelegt werden.

- Selektierte importieren

Startet den Import aber importiert nur die Buchungen, die in der Liste selektiert wurden. Zum Auswählen mehrerer Einträge kann die Maustaste zusammen mit STRG oder Shift geklickt werden. Nachdem die Verarbeitung abgeschlossen ist, erhalten Sie entweder eine Erfolgsmeldung oder eine Liste der aufgetretenen Fehler.

- Selektierte importieren

Startet den Import aller Einträge der CSV-Datei. Nachdem die Verarbeitung abgeschlossen ist, erhalten Sie entweder eine Erfolgsmeldung oder eine Liste der aufgetretenen Fehler.

- Zurück

Beendet den Dialog, ohne den Import zu starten. Alle Änderungen an der Importdefinition werden jedoch gespeichert.

 

Dialog Neue Importbeschreibung

Geben Sie bitte hier den Namen für die neue Importbeschreibung ein und bestätigen mit OK. Mit dem Schalter Abbrechen gelangen Sie in den Dialog Import zurück ohne das eine neue Importbeschreibung angelegt wird.

Die Option Vorhandene Eigenschaften übernehmen übernimmt alle Einstellungen (Dateinamen, Trennzeichen, Zuordnung der Buchungsfelder zu Dateifeldern, etc) aus der gerade ausgewählten Importbeschreibung in die neu erstellte.

Dialog Protokoll

Sie erhalten hier eine Aufstellung aller Fehler, die während des Imports aufgetreten sind.

Fehlermeldungen

Hier werden alle Fehler mit einer kurzen Beschreibung aufgelistet. Soweit vorhanden wird die Zeile, in welcher der Fehler auftrat angezeigt.

 

Schrittweise Anleitung

Mit Hilfe der folgenden schrittweisen Anleitung können Sie eine neue Importbeschreibung anlegen.

 

1.        Klicken Sie auf die Schaltfläche Neu. Der Dialog Neue Importbeschreibung wird geöffnet. Hier können Sie einen Namen für die neue Importdefinition eingeben. Bestätigen Sie Ihre Eingabe mit OK.

2.        Geben Sie im Feld Dateiname den vollständigen Pfad- und Dateinamen der zu importierenden Datei an. Über die Schaltfläche Durchsuchen... können Sie eine Datei auswählen.

3.        Ändern Sie gegebenenfalls im Feld Trennzeichen das Zeichen, mit dem die Felder in der oben angegebenen Datei voneinander getrennt sind.

4.        Sofern die Datei Kopfzeilen enthält (z. B. Feldnamen) können Sie deren Anzahl im Feld Anzahl Kopfzeilen angeben.

5.        Klicken Sie jetzt auf die Schaltfläche Anzeigen. Die Liste Dateiinhalt wird aktualisiert. Sie sollten die ersten zehn Zeilen der zu importierenden Datei sehen. Überprüfen Sie bitte, ob die dort angezeigten Feldinhalte plausibel erscheinen.

6.        Geben Sie nun in den Feldern im Kasten Zuordnung für jedes Buchungsfeld an welchen Feldinhalt es aus der zu importierenden Datei erhalten soll. Lesen Sie hierzu bitte die Erläuterungen im Kapitel Dialog ImportIDH_DLGIMPORT.

Über die Schaltfläche Start können Sie nun die Daten importieren. Die Schaltfläche Schließen beendet den Dialog ohne die Daten zu importieren, alle Änderungen an der Importbeschreibung bleiben jedoch erhalten.

 

Arbeiten mit GAWK-Skripten

AWK ist eine in Bezug auf Zeichenkettenmanipulation extrem mächtige und effiziente Skriptsprache. Das mit dem Plugin mitinstallierte GAWK.exe ist die GNU-Implementierung der Programmiersprache AWK und wird von der Free Software Foundation gepflegt.

 

Wozu benötigt man Skripte? Selten kommen die CSV-Daten in einer Form vor, die EC&T ohne weitere Ergänzung 1:1 verarbeiten kann. Das ist nur der Fall, wenn man einen EC&T-Export (siehe Datei-Menü) re-importiert. Z.B. wenn man eine CSV-Datei von seinem Bankkonto hat, fehlt dort natürlich der MWSt.-Satz. GAWK erlaubt es Regeln zu formulieren, mit denen man diese Information ergänzen kann und minimiert den Aufwand für die manuelle Bearbeitung erheblich. GAWK kann also Felder (bzw. Spalten) in die CSV-Datei einfügen, z.B. Belegnummer, die vorher, um bei dem Beispiel Bankkonto zu bleiben, aus einem Verwendungszweck-Feld herausextrahiert wurde – oder aber Felder zusammenfassen, z.B. wenn mehrere Verwendungszweck-Felder vorhanden sind oder man den Kontoinhaber des Gegenkontos und das Verwendungszweckfeld verschmelzen möchte.

 

Wie arbeitet GAWK? GAWK verarbeitet die Eingabedatei (die CSV-Datei) zeilenweise und zerlegt dabei die Felder entsprechend dem eingestellten Trennzeichen. Diese Felder kann man im Skript mit $1, $2, $3 usw. ansprechen. $0 repräsentiert die gesamte Eingabezeile. Das einfachste GAWK-Skript sieht so aus und macht nichts anderes als die ursprüngliche Datei zu kopieren:

 

{ print $0 }

 

Möchte man nur die 3. und 4. Spalte aus einer Datei extrahieren geht das folgendermaßen (Semikolon als Trennzeichen):

 

                { print $3 ";" $4 }

 

Korrekter wäre es hier, statt des ";" ein FS einzufügen. Das steht für Feld-Separator. FS wird vom Import-Plugin automatisch gesetzt und entspricht immer dem im Import-Dialog angegebenen Trennzeichen. Es gibt noch andere voreingestellte Platzhalter, etwa NF (number of fields) der der Anzahl der Felder der Zeile entspricht. $NF würde also das letzte Feld der Zeile bezeichnen.

 

Die geschweiften Klammern { } bedeuten das hier ein oder mehrere Befehle enthalten sind, die unbedingt auf jede Zeile anzuwenden sind. Stände vor dem { eine Bedingung, etwa NF >= 4, würden die in den Klammern befindlichen Befehle nur ausgeführt, wenn die Bedingung erfüllt ist (in diesem Fall, wenn die Anzahl der Felder größer oder gleich 4 ist).

 

Für Operationen, die nur am Anfang oder Ende der Verarbeitung ausgeführt werden sollen, gibt es die Schlüsselwörter BEGIN und END. Z.B. Kann man mit BEGIN vor der Verarbeitung der Eingabedatei eine Kopfzeile ausgeben.

 

Einen Schönheitsfehler hat GAWK für deutsche und österreichische Benutzer: Die interne Fließkommafunktion arbeitet mit dem Dezimalpunkt. Möchte man also Berechnungen anstellen, muss das Komma immer erstmal konvertiert werden, z.B.:

 

{

                netto = $7;                                            # Netto-Betrag steht im 7. Feld

                sub(/,/, ".", netto);                                              # Komma durch Punkt ersetzen

                brutto = netto * 1.19;                           # Brutto-Betrag errechnen

                brutto = sprintf("%.2f", brutto + 0.005);          # auf 2 Stellen nach dem Komma kürzen, dabei

# kaufmännisch runden

                sub(/\./, ",", brutto);                             # jetzt den Dezimalpunkt wieder durchs Komma ersetzen

                print netto ";" brutto                          # Ausgabe des Ergebnisses

}

 

In den sub Funktionen (steht für Substitution, Ersetzen) ist der erste Parameter ein sogenannter regulärer Ausdruck (auf englisch: regular expression). Damit können komplexe Suchmuster angegeben werden. Suchte ich also eine 5-stellige Rechnungsnummer in einem Verwendungszweck-Feld, würde das so aussehen:

 

BEGIN { print "Verwendungszweck;Belegnummer" ; anzahl = 0 }

{

                position = match($10, /R[0-9][0-9][0-9][0-9][0-9][^0-9]/)

                if (position > 0)

{

                               belegnummer = substr($10, position, 6);

anzahl = anzahl + 1;

}

                else

                               belegnummer = "keine gefunden";

                print $10 ";" belegnummer

}

END { print ";----------------------------------\nAnzahl gefundener Rechnungsnummern;" anzahl }

 

match() gibt die Position der dem Muster /R[0-9][0-9][0-9][0-9][0-9][^0-9]/ entspechen Zeichenkette wieder (ein R mit 5 Ziffern, gefolgt von einem Zeichen, das keine Ziffer ist); wenn diese nicht gefunden wurde ist position gleich 0 und es wird "keine gefunden" in die Belegnummernspalte eingetragen, ansonsten wird die 6-stellige Zeichenkette an der gefundenen position extrahiert und ausgegeben. Der Code veranschaulicht außerdem die Nutzung von BEGIN und END.
